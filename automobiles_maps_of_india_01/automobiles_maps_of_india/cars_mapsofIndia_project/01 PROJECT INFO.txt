- And finally, run the application: python manage.py runserver

**------Automobile-company------**


**1. Introduction**
**1.1 Purpose**
-The purpose of this document is to outline the requirements for the automobile_company project, which is a Django-based web application focused
 on presenting automobile industry data, company information, historical articles, car models, and related information.

**1.2 Scope**
-The project includes functionalities for displaying automobile companies, car manufacturers, commercial vehicles, historical articles, car
 models, and lubricants. It provides a responsive web interface for users to explore data and information about the automobile industry.

**1.3 Definitions, Acronyms, and Abbreviations**

**DFD: Data Flow Diagram**
  -SRS: Software Requirements Specification
  -CRUD: Create, Read, Update, Delete
**2. System Overview**
-The system provides an interface for users to view and interact with:

-Automobile companies and their profit data
-Car manufacturers and their models
-Commercial vehicles and their specifications
-Historical articles related to automobiles
-Lubricants and additives used in vehicles
**3. Functional Requirements**
-3.1 View Automobile Companies

--Display a list of automobile companies in India.
--Show profit data for the selected year.
--Provide quick links to commercial vehicles, car manufacturers, and two-wheeler manufacturers.
**3.2 Home Page**

-Display a summary of the top 3 companies, vehicles, and latest news items.
**3.3 Car Manufacturers**

-List car manufacturers in JSON format.
**3.4 Commercial Vehicles**

-Display information about commercial vehicle manufacturers and their vehicles.
**3.5 Automobile History**

-List all historical articles.
-Display detailed information about a specific article, including related car models and navigation to previous/next articles.
**3.6 Lubricants and Additives**

-Display information about lubricants and additives.
**4. Non-Functional Requirements**
**4.1 Performance**

-The system should handle a high volume of users and data efficiently.
**4.2 Usability**

-The web interface should be user-friendly and responsive.
**4.3 Security**

-Ensure secure access to data and prevent unauthorized access.

**5. System Architecture**
**5.1 Overview**
-The system is built using Django and follows the Model-View-Template (MVT) architectural pattern.

**5.2 Components**

-Models: Define the data structure and relationships.
-Views: Handle requests and return responses.
-Templates: Render HTML for the user interface.
